- 列表延续暖白背景、橄榄色层级、细分隔线、素色橙色主动作和轻量横向菜谱行。原生状态栏与导航栏解释了相对参考图的纵向偏移。
- 编辑器保留草稿标题、保存状态、五步进度、分隔线表单、步骤排序动作和底部固定导航。实现额外展示产品规格要求的“烹饪方式（可选）”，因此正文比参考图更长，可继续纵向滚动；这不是内容裁切。
- 已审核图片页使用唯一已核验的 CC0 番茄炒鸡蛋照片，并展示作者、许可、来源页、获取日期和复制来源入口。客户端只加载服务端返回的已审核资产。
- 375、390、430px 均未发现横向溢出、底部安全区遮挡、主按钮裁切或当前菜名截断。

## 原生交互证据

- 新建并重新打开“番茄炒蛋”草稿，填写基本信息后看到“已保存”。
- 把预计分钟从 15 改为 16 后立即返回列表，再次打开时服务端返回 16，证明页面隐藏时的强制 flush 生效；随后恢复为 15。
- 添加番茄和鸡蛋，番茄数量留空时预览显示“适量”，鸡蛋显示“2枚”。
- 添加两条做法并通过原生“下移”操作把顺序改为“锅中炒匀”在前、“盛盘即可”在后；预览与无障碍树顺序一致。
- 从已审核图库选择图片后，编辑器展示照片及作者、许可、获取日期；预览同时展示图片、食材与排序后的步骤。
- 真实点击“发布菜谱”调用本地 API，服务端返回 HTTP 503；页面明确显示“暂时无法完成安全检查，请稍后重试”，发布按钮可重试且草稿仍保留。没有把该结果写成发布成功。
- 切换设备导致模拟器存储隔离后，重新登录仍能取回同一服务端草稿；最终 390px 复验显示开发者工具应用错误 0、问题 0。其余信息为本地 HTTP 图片、基础库灰度、热重载或平台弃用警告。

## 本轮修复

1. P1：430px 的家庭菜谱行受原生 `<button>` 固有居中/收缩行为影响，图片、文字和动作被压成窄列。先新增失败契约测试，再为 `.recipe-row` 增加明确的 `min-width: 100%` 和 `justify-content: flex-start`；375、390、430px 原生复验均恢复为完整横向行。

修复并重新比较后，本范围没有剩余 P0、P1 或 P2 视觉缺陷。参考图与原生实现之间的状态栏、导航栏和规格新增字段差异均有明确平台或产品原因。
